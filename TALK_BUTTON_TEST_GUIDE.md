# Talk Button Testing Guide

## What You Have Now ✅

Looking at your screenshot, you have the **perfect PTT UI** showing:
- ✅ **User** - Sender name
- ✅ **Walkie-Talkie** - Channel name  
- ✅ **Leave** button - To dismiss
- ✅ **Talk** button (🎙️) - To reply

This UI is generated by the **PTT Framework** (iOS 16+) and appears when you return `.activeRemoteParticipant` from `incomingPushResult`.

## How the Talk Button Works

### When You Press Talk Button:

```
User presses Talk button 🎙️
    ↓
iOS PTT Framework calls:
channelManager(_:channelUUID:didBeginTransmittingFrom:)
    ↓
Get groupId (from currentGroupId or UserDefaults)
    ↓
Call NativePTTPlayer.startTransmitting(groupId)
    ↓
Connect WebSocket (if needed)
    ↓
Start recording audio chunks (1.5s each)
    ↓
Send chunks to server via WebSocket
    ↓
User releases Talk button
    ↓
iOS PTT Framework calls:
channelManager(_:channelUUID:didEndTransmittingFrom:)
    ↓
Stop recording and send final chunk
    ↓
✅ Your voice is delivered to other user!
```

## Troubleshooting

### Problem 1: Talk Button Doesn't Appear

**Symptoms**:
- PTT UI shows but no Talk button visible
- Or entire UI doesn't appear

**Cause**: PTT Framework not showing interactive UI

**Solution**:
The Talk button should be there in the UI you showed in the screenshot. If it's grayed out:
1. Make sure `didActivate` was called
2. Check that audio session is active

### Problem 2: "Cannot transmit: No groupId available!"

**Symptoms**:
```
❌ Cannot transmit: No groupId available!
💡 Tip: Make sure a PTT message was received before trying to reply
```

**Cause**: `currentGroupId` was not set when PTT push arrived

**Solution**:
1. Make sure you received a PTT message FIRST
2. Check that `incomingPushResult` set the `currentGroupId`:
   ```
   📨 PTT Push Received: ["groupId": xxx, ...]
   ```
3. Try pressing Talk button IMMEDIATELY after audio finishes (before session times out)

### Problem 3: No Audio Chunks Sent

**Symptoms**:
```
🎙️ Began Transmitting
...
(No "Sent audio chunk" logs)
```

**Causes**:
1. WebSocket not connected
2. userId not in UserDefaults
3. Microphone permission denied

**Solutions**:
```
# Check userId is stored
Look for: 👤 User ID: ajaw9LhcwUSp5tyoVXorVYV8N473

# Check WebSocket
Look for: ✅ NativePTTPlayer: WebSocket connected
OR: ♻️ Reusing existing WebSocket connection

# Check microphone permission
Settings > MariSpeak > Microphone > Enable
```

### Problem 4: Other Device Doesn't Receive Reply

**Symptoms**:
- Device A sends successfully
- Device B doesn't get the message

**Check Server Logs**:
```
# Should see:
✅ Registered: ajaw9LhcwUSp5tyoVXorVYV8N473
Audio from ajaw9LhcwUSp5tyoVXorVYV8N473 to group xxx
📤 Forwarding to connected users
```

**Solutions**:
1. Check both devices are on same group
2. Check Device B is connected to server
3. Check Device B is not in background (if it is, should trigger VoIP push)

```bash

xcrun simctl spawn booted log stream --predicate 'process contains "Runner"' | grep "User ID"
```

### Check Server is Running
```bash
lsof -i :3010
# Should show node process listening
```

### Monitor Server Logs
```bash
# In your server directory:
tail -f ptt-server.log
# Or if logging to console:
node ptt-server.js
```
